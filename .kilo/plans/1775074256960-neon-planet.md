## 11. PROBLEMA REPORTADO: QR NO VISIBLE

### Síntoma
El usuario reporta que los códigos QR no son visibles al intentar generarlos.

### Análisis del Problema

El código actual en `qr_service.py` tiene posibles problemas:

1. **Biblioteca qrcode requiere Pillow**: Para generar imágenes, qrcode necesita Pillow instalado
2. **API de qrcode puede haber cambiado**: El uso de `qrcode.constants.ERROR_CORRECT_L` puede no funcionar
3. **El método `img.save()` con format="PNG"** puede fallar si Pillow no está instalado

### Plan de Solución

#### Paso 1: Verificar dependencias
```bash
pip list | grep -E "qrcode|Pillow"
```

#### Paso 2: Corregir código en `backend/services/qr_service.py`

Cambiar el método `generar_qr_base64` para:
1. Usar Pillow directamente para generar la imagen
2. Manejar errores de forma más robusta
3. Proporcionar fallback si no se puede generar

#### Paso 3: Código corregido propuesto

```python
@staticmethod
def generar_qr_base64(data: str, size: int = 200) -> str:
    try:
        import qrcode
        from PIL import Image
        
        qr = qrcode.QRCode(
            version=None,  # Auto
            error_correction=qrcode.constants.ERROR_CORRECT_L,
            box_size=10,
            border=4,
        )
        qr.add_data(data)
        qr.make(fit=True)
        
        img = qr.make_image(fill_color="black", back_color="white")
        
        # Resize if needed
        if size != 200:
            img = img.resize((size, size), Image.LANCZOS)
        
        # Convertir a bytes
        buffer = io.BytesIO()
        img.save(buffer, format="PNG")
        buffer.seek(0)
        
        # Convertir a base64
        img_base64 = base64.b64encode(buffer.getvalue()).decode()
        return f"data:image/png;base64,{img_base64}"
        
    except ImportError as e:
        print(f"Error de importación: {e}")
        return ""
    except Exception as e:
        print(f"Error generando QR: {e}")
        return ""
```
